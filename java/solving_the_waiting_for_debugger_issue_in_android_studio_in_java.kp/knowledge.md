# Causes
Android Studio may display the message "Waiting For Debugger" in the console if the debugger is not properly attached to the application. This could be caused by:

1. Incorrectly configured run/debug configuration
2. Debugger is not attached to the application

# Symptoms
When the debugger is not attached to the application, the following symptoms may be observed:

1. The application will not respond to any user input
2. The application will not respond to any breakpoints
3. The application will not respond to any debugging commands
4. The Android Studio console will display the message "Waiting For Debugger"

# Solutions
The following steps can be taken to resolve the issue:

1. Ensure that the run/debug configuration is correctly configured
2. Attach the debugger to the application
3. Restart the application
4. Check that the debugger is correctly attached to the application
