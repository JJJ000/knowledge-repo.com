# Section 1: Repository Size

GitHub.com allows users to store up to 100GB of data in a single repository. This includes all of the files in the repository as well as any assets associated with the repository, such as images and videos.

# Section 2: File Size

GitHub.com also limits the size of individual files that can be uploaded to the repository. The maximum file size is 25MB. Files larger than this will not be accepted.

# Section 3: Number of Files

GitHub.com does not limit the number of files that can be stored in a repository. However, it is recommended that repositories contain fewer than 10,000 files in order to ensure optimal performance.

# Section 4: Number of Commits

GitHub.com does not limit the number of commits that can be made to a repository. However, it is recommended that repositories have fewer than 1,000 commits in order to ensure optimal performance.
