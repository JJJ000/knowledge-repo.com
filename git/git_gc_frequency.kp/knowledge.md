#### Frequently

Git-gc should be run frequently in order to keep the repository optimized. This should be done at least once a month, as well as after large pushes or merges.

#### After Big Changes

Git-gc should be run after any major changes to the repository, such as a large push or merge. This will help keep the repository optimized and ensure that any changes are reflected in the repository.

#### After Adding Large Files

Git-gc should also be run after adding large files to the repository. This will help keep the repository optimized and ensure that any changes are reflected in the repository.

#### After Deleting Files

Finally, git-gc should be run after deleting files from the repository. This will help keep the repository optimized and ensure that any changes are reflected in the repository.
